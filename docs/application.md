# Desktop Application

The ShowControl desktop app has five main tabs:

- **Info**
- **Setlist**
- **Performance**
- **User**
- **Settings**

The typical workflow is:

1. Configure device/network in **Settings**
2. Build and verify content in **Setlist**
3. Validate mappings in **User**
4. Use **Performance** during rehearsal/show

---

## Main navigation

The top bar is your global control area:

- main tab switching
- USB status indicator
- Ethernet status indicator

### What to verify first

- at least one active connection path (USB or ETH)
- expected tab selected for your current task

---

## Setlist tab

Setlist is the primary preparation workspace.

### Main areas

- **Left column**: source content (Scenes/Markers)
- **Right column**: Songs/Sections list
- **Top controls**:
  - undo/redo
  - setlist selector
  - new/rename/duplicate/delete setlist
  - total setlist duration

### Typical setlist workflow

1. Choose source (**Scenes** or **Markers**)
2. Create or select target setlist
3. Drag items into Songs
4. Organize Sections within each Song
5. Reorder for final show flow
6. Validate duration and transitions

### Good practice

- keep Song names performance-friendly
- keep Sections meaningful for jumps/loops
- use undo/redo often during arrangement

---

## Performance tab

Performance is a focused live-use view.

Use it for fast access during rehearsal and show operation when you no longer need deep editing.

---

## User tab

User tab defines controller behavior for each **User page**.

### You can configure

- button actions (normal + shifted)
- encoder turn/push actions
- pedal behavior
- display line content

### Recommended validation pass

For each User page used in your show:

1. Test critical transport actions
2. Test next/previous Song behavior
3. Test shifted actions
4. Test pedals and encoder

---

## Settings tab

Settings combines visual hardware settings and network communication settings.

### Device settings

- LED brightness
- display brightness

### Network settings

- IP address (4 fields)
- port
- send/apply action

### Notes

- keep production network values documented
- avoid changing network values during a live section

---

## Info tab

Info is the built-in documentation viewer.

- left column: manual navigation
- right pane: current page content
- version label: quick app version reference

---

## Best-practice flow before a show

1. Confirm USB/ETH status
2. Confirm Settings values
3. Finalize setlist order and durations
4. Validate User page mappings
5. Rehearse key transitions in Performance

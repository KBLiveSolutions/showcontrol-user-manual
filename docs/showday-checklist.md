# Show Day Checklist

Operator-first checklist for rehearsal and live runtime.

---

## Pre-show (T-30 to T-10)

- [ ] Controller powered and physically secured
- [ ] USB/ETH route validated
- [ ] App running on correct machine/profile
- [ ] Correct setlist loaded
- [ ] First Song selected
- [ ] Critical mappings validated (transport, Song nav, Shift)

---

## Line-check (T-10 to T-2)

- [ ] Play/Stop test
- [ ] Previous/Next Song test
- [ ] Section jump test
- [ ] Shift-layer test
- [ ] Pedal + encoder test

---

## During show (operator rules)

- Do not change IP/port mid-song.
- Do not remap controls mid-song unless emergency.
- Keep fallback path ready before each transition.

---

## Fallback protocol

1. Verify active transport path (USB/ETH indicator).
2. Verify selected setlist and current Song.
3. Execute known-safe action (e.g., Play/Stop check).
4. If unstable, restart app at safe transition point.
5. Re-validate first actionable control before resuming.

---

## Post-show capture

- What failed?
- What took too long?
- Which action needs a dedicated control?
- Which mapping should be simplified?

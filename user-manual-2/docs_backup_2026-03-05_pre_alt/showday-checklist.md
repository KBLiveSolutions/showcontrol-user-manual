# Show Day Checklist

Use this page during rehearsal and before going on stage.

## A) Pre-show (T-30 to T-10)
- [ ] Controller connected (USB and/or ETH)
- [ ] ShowControl app open on correct machine
- [ ] Correct project/open set confirmed
- [ ] Brightness adjusted for venue lighting
- [ ] Correct setlist selected
- [ ] First song highlighted
- [ ] Critical buttons and pedals tested

## B) Line-check (T-10 to T-2)
- [ ] Transport test (Play/Stop)
- [ ] Previous/Next song test
- [ ] Section jump test
- [ ] Shift-layer sanity check
- [ ] Encoder test
- [ ] MIDI/OSC feedback observed

## C) During show
- Keep one hand rule for critical actions.
- Avoid editing mappings mid-show unless essential.
- If behavior is inconsistent, use the fallback protocol below.

## D) Emergency fallback protocol

### If setlist/source does not respond
1. Confirm correct tab and selected setlist.
2. Try one known-safe action (Play/Stop).
3. Re-check USB/ETH indicators.

### If Ableton/script path fails
1. Keep controller on stable local actions.
2. Use backup navigation path (manual selection / alternate trigger).
3. Reconnect script path at safe transition point.

### If app UI becomes inconsistent
1. Save state if possible.
2. Restart app.
3. Reconfirm setlist and first active song.

## E) Post-show notes (for next rehearsal)
- What failed?
- What was slow?
- What needs remapping?
- What should become a one-button action?
